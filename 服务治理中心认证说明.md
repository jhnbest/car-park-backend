## **身份认证和权限**

服务治理中心的服务，都需要对服务请求者进行身份认证。对于请求一个服务，其调用过程相对复杂，涉及到身份、认证、权限三个概念。

## **身份认证**

其实就是网关通过对请求者凭证的识别来达到认证请求者身份的目的。在 OpenAPI 的场景里，我们主要是用hamc来作为服务请求者身份认证的方式，hmac是以一对 Access Key Id（应用组件的appId） 和 Access Key Secret 经过算法计算得到摘要进行匹配。
除了AK/SK摘要 外，服务治理中心还提供一个Date字段的时间戳， 因为具有时效性，它具有更高级别的安全保障。每一笔服务请求的header 由 Access Key Id/Access Key Secret的摘要+ Date时间戳组成。

其中：

Date：Wed, 13 Sep 2023 07:11:21 GMT

注意：如果有上游服务自有身份认证字段也是header中的Authorization同名，则可以将服务治理中心的Authorization参数值设置为：Proxy-Authorization

## **签名机制**

在实际的 OpenAPI 调用中，不会将 Access Key Secret 发布给网关的形式来进行认证，而是采用非对称加密的形式来对请求的内容进行签名。网关会根据获取的 Access Key Id 获得对应的 Access Key Secret，并对收到的请求进行签名，通过对比签名是否一致，从而判断凭证的有效性完成认证。

public static byte[] signatureReturnBytes(String data, String key) throws NoSuchAlgorithmException, InvalidKeyException {

SecretKeySpec signingKey = new SecretKeySpec(key.getBytes(), "HmacSHA256");

Mac mac = Mac.getInstance("HmacSHA256");

mac.init(signingKey);

return mac.doFinal(data.getBytes());

}

public static String encryptPassword(String password){

byte[] hashBytes = sha(password);

return Base64.encodeBase64String(hashBytes);

}

public static byte[] sha(final String strText) {

if (strText != null && strText.length() > 0) {

try {

MessageDigest messageDigest = MessageDigest.getInstance("SHA-256");

messageDigest.update(strText.getBytes());

return messageDigest.digest();

} catch (NoSuchAlgorithmException e) {

return null;

}

} else {

return null;

}

}
